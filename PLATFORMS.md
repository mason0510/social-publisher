# 平台支持说明

## 🌍 定位：国际社交媒体发布工具

Social Publisher 专注于**国际社交媒体平台**的内容发布和自动化。

### 为什么将国内外平台分离？

国内社交媒体平台（微博、小红书、抖音等）与国际平台在以下方面存在显著差异：

1. **认证机制**
   - 国内：复杂的手机验证、实名认证、图形验证码
   - 国际：相对标准的 OAuth 2.0 / API Key

2. **内容审核**
   - 国内：严格的内容审核、敏感词过滤
   - 国际：相对宽松的内容政策

3. **API 可用性**
   - 国内：API 限制多，部分平台无公开 API
   - 国际：API 文档完善，开发者友好

4. **技术实现**
   - 国内：需要浏览器自动化（Playwright/Puppeteer）
   - 国际：可直接使用 REST API

因此，我们将国内平台由**独立的 CLI 工具**处理（如 `weibo-cli`、`xhs-cli` 等），保持代码简洁和维护性。

## 🇨🇳 国内平台（独立工具）

国内平台由以下独立 CLI 工具处理：

| 类型 | 平台 | CLI 工具 | 状态 |
|------|------|---------|------|
| **技术内容** | 掘金/CSDN/知乎 | `tech-publish` | ✅ 已实现 |
| **短视频** | 抖音 | `douyin-cli` | ✅ 已实现 |
| **社交电商** | 小红书 | `xhs-cli` | ✅ 已实现 |
| **社交媒体** | 微博 | `weibo-cli` | 📝 规划中 |
| **视频号** | 微信视频号 | `wechat-channels-cli` | 📝 规划中 |

**为什么国内平台需要独立工具**：
- 🔐 更复杂的认证和验证机制（手机验证、实名认证、图形验证码）
- 📝 严格的内容审核要求（敏感词过滤、人工审核）
- ⚠️ API 受限或无公开 API
- 🤖 需要浏览器自动化（Playwright/Puppeteer）
- 🏗️ 技术栈与国际平台差异大

**现有工具说明**：
- `tech-publish`: 专注于技术内容平台（掘金、CSDN、知乎），适合技术文章发布
- `douyin-cli`: 抖音短视频发布和管理
- `xhs-cli`: 小红书图文和短视频发布
